Identificação de Transações Fraudulentas

Usei o modelo do Classificador LGBM para aprender todo o comportamento dos dados e, após modelar os dados, usei-o para fazer previsões, usando uma amostra de 20% do conjunto de dados completo para evitar erro de memória. Depois disso, dividi em 70% do conjunto de dados para treiná-lo e depois testei em 30% dos dados não vistos pelo modelo para medir seu desempenho. Após todos os passos citados e com as previsões feitas, realizei a avaliação do desempenho do modelo, trazendo o resultado da classificação para cada transação, através de quatro métricas: precisão, exatidão, recall, f1-measure. Trouxe também 3 gráficos que demonstram o resultado e erro do modelo para transação e por fim consegui gerar um resultado do modelo com uma perda total economizada de R$ 34,50.000 e 89% de detecção de fraude na transação.

As ferramentas utilizadas foram:

  • Python, Pandas, Numpy e Seaborn
  • Scikit-learn
